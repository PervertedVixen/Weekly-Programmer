Weekly Programmer - Week 12 =========================== You are in a Maze of Twisty Passages, all alike ----------------------------------------------- From [Weekly Programmer Challenges](./index.html) Description ----------- Write a program that accepts a rectangular maze and solves it using any maze solving algorithm of your choosing. A valid maze will have an entrance on the top edge and an exit on the bottom. A maze may have loops, and may include inaccessible regions. Mazes will be input as text where non-space characters indicate a maze wall and the space character indicate a maze corridor For Example, observe the following grid. ```text ┃ ━━━━━━┳━┓ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┃ ┣━┫ ┗━╋━┻━┫ ┃ ┃ ┃ ┃ ┃ ┗━━ ┃ ┃ ┃ ┃ ┃ ┃ ┗━━━━━━━┛ ┃ ``` It is a value maze of size 11, and after parsing and solving it might produce the following output: ```text ┃#━━━━━━┳━┓ ┃### ┃ ┃ ┃ ┃#┃ ┃ ┃ ┃ ┃ ┃#┃ ┃ ┃ ┃ ┃#┃ ┃ ┃ ┃ ┃ ┃#┃ ┃ ┃ ┃ ┣━┫#┗━╋━┻━┫ ┃ ┃###┃###┃ ┃ ┗━━#┃#┃#┃ ┃ ###┃#┃ ┗━━━━━━━┛#┃ ``` Formal Inputs & Outputs ----------------------- ### Input Description Mazes will be input as a string where each row of the maze is separated by new line characters and each character in a row represents a cell which may be traversable (a corridor) or untraverseable (a wall). A valid maze will have an entrance on the top edge and an exit on the bottom. A maze may have loops, and may include inaccessible regions. Mazes will be input as text where non-space characters indicate a maze wall and the space character indicate a maze corridor ### Output Description You should ouput a standardized representation of the maze drawn with ASCII art, using one character of your chose for the walls, a different character of your choice for the untraveled corridors and a final character of your choice for traversed corridors. *Alternatively:* You may construct the maze walls using the Unicode variants of the old Extended ASCII box drawing characters, should you so desire.
